# Neuro Nutri Clinic — vercel/ app

- The whole frontend is one state-machine SPA: `vercel/src/components/mockups/nnc-site/Site.tsx`. `currentScreen` state toggles screens; no router. `ScreenHome` is the public landing page; the other screens (enroll/intake/consent/booking/confirmation/login/signup/dashboard/admin) are the **functional booking/auth flow** that POST to `/api/enroll`, `/api/auth/*`, `/api/my/*`, `/api/admin/*`. Redesign the homepage freely but do not touch the functional screens' logic.
- Nav anchor links rely on section `id`s in ScreenHome: `philosophy`, `services`, `journey`, `about`, `faq`. Keep these ids when rewriting the homepage or nav scrolling breaks. CTAs navigate via `onNavigate("enroll")`.

## Tailwind v4 + brand colors (important)
- The project uses **Tailwind v4** (`@import "tailwindcss"` + `@tailwindcss/vite`); there is **no `tailwind.config.*`**.
- The `nnc-*` brand palette is registered in `@theme` in `vercel/src/index.css`. **Why:** without registering colors there, utilities like `bg-nnc-sage/60`, `from-nnc-blush/20`, `border-nnc-rose` silently do not generate (no error, just missing styles). Registering `--color-nnc-*` in `@theme` makes every opacity/gradient/border variant work automatically.
- `_group.css` holds non-color custom utilities only worth keeping: `font-serif/sans/script`, `shadow-nnc-soft`, `shadow-nnc-glow`, `nav-underline`, `bg-gradient-botanical`, `blob-shape`, `blob-shape-2`, `hover-lift`. The per-variant `nnc-*` color classes there are now redundant (generated by @theme) but harmless.
- Brand hex: cream #F3FBF7, ivory #FDFDFD, sage(mint) #3CC393, olive(fuchsia) #C42A9B, blush #F7DBEE, rose #E284C2, charcoal #322A30. Fonts: Cormorant Garamond (serif) / Inter (sans) / Caveat (script).
